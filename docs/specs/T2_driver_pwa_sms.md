# T2: Driver PWA Bootstrap + Outbound Dispatch SMS

> **Retroactive spec.** Reverse-engineered from shipped code and `supabase/migrations/NOTE_T2_APPLICATION_CODE.md`; format per `docs/specs/TEMPLATE.md`.

## Purpose

When a dispatch is assigned to a driver, the platform sends an outbound SMS bootstrap link. The driver opens it on their phone, hits `/api/driver/session` to establish an opaque-token session, and from there pings location, advances the phase machine, and fetches available loads. The outbox worker cron processes queued SMS events idempotently.

## Tables touched

Migrations `031`–`034` and `036`–`037`. (`035` intentionally skipped — no number reuse.)

- `driver_sessions` (new, `031_driver_sessions.sql`) — opaque-token session store, sha256 at rest
- `location_pings` (new, `032_location_pings.sql`) — time-series of GPS pings, `anomaly_flags` array
- `geofences` (new, `033_geofences.sql`) — polygon store for site/route fences
- `sms_consent` (new, `034_sms_consent.sql`) — TCPA consent grants per phone
- `storage.objects` policies (modified, `036_dispatch_media_bucket_rls.sql`) — 8 policies scoping the `dispatch_media` bucket to service/admin/driver/contractor/supplier roles
- `outbox_events` (modified, `037_outbox_pending_index.sql`) — adds `outbox_events_pending_created_at_idx` for the worker claim query

## API surface

All under `src/app/api/`:

- `POST /api/driver/session` — body validated by Zod; on success sets HttpOnly `em_driver_session` cookie; 400 on empty/bad body
- `POST /api/driver/ping` — cookie-auth; inserts into `location_pings`; sync anti-spoof gates (velocity > 85 mph, teleport > 5 km in < 30 s, clock_skew > 60 s) populate `anomaly_flags`; expensive gates deferred
- `GET /api/driver/available-loads` — cookie-auth; driver's eligible dispatch pool
- `POST /api/driver/advance-phase` — cookie-auth; state machine step via `src/lib/driver/phase-machine.ts`
- `POST /api/driver/location` — cookie-auth; related location write path
- `GET /api/cron/outbox-worker` — Vercel cron (every minute via `vercel.json`); atomic claim + send; idempotent

## Acceptance criteria

- [ ] Outbound bootstrap SMS is generated by the correct template and reaches the driver phone via Twilio.
- [ ] `/api/driver/session` fails closed when `sms_consent` lacks the `dispatch_plus_location` grant for the phone.
- [ ] Outbox worker claim-and-send is atomic: two concurrent runs never double-send the same event.
- [ ] `location_pings.anomaly_flags` is always populated — `'{}'` when clean, otherwise the detected flags.

## Known TCPA gap

Until the inbound state machine ships, Twilio inbound webhook URL must be blank in the console. STOP/HELP auto-replies from Twilio's opt-out compliance keyword list still work at the carrier layer, but app-side consent state transitions are not recorded.
